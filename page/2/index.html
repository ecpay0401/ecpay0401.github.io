綠界-金流 常見付款狀態未更新原因

發表於 2020-09-07 分類於 金流

常見付款狀態未更新原因
若消費者已經付款，綠界廠商管理後台也已更新付款

但您的系統後台(Woocommerec、Opencart、Shopline、QDM、美安….平台)

付款狀態未被更新，請程式人員檢查以下設定:

(1)接收付款通知的伺服器是否異常

● 如您正在舉辦搶購活動，請務必確保伺服器可短時間大量接收綠界回傳的付款結果通知

Ex:AWS、台灣固網….等其他網路業者，預設防火牆機制，短期間接收大量回傳會視為攻擊事件，請將綠界IP加入白名單

(2)近期是否有變更接收付款通知的ReturnURL網址

(3)接收付款結果的ReturnURL程式是否有判斷交易成功(RtnCode=1表示付款成功，其餘代碼為失敗)

(4)貴司系統收到綠界回傳結果，是否有正確回應1|OK

(5)因資安政策規定綠界僅提供443 PORT且需使用合法的DomainName進行設定

上述原因確認完畢，仍然無法排除此問題請提供以下資訊

並Email至techsupport@ecpay.com.tw

(1)商店代號(MerchantID)

(2)廠商交易編號(MerchantTradeNo)

(3)交易日期(MerchantTradeDate)

綠界-金流 綠界帳戶餘額低於退刷金額或為負數導致無法退刷及建立物流訂單

發表於 2020-09-07 更新於 2020-09-14 分類於 金流

若您的帳戶餘額低於退刷金額為負數

將無法進行退刷以及建立物流訂單

請至綠界廠商管理後台進行預付動作，

如需進行退刷作業，服務項目請選擇”信用卡退刷款” 請存入足夠退刷的金額

如需進行物流費用預存，服務項目請選擇”預付物流款”請存入足夠每周二結算物流費用使用金額。

綠界-物流 電子地圖無法顯示

發表於 2020-09-05 更新於 2020-11-26 分類於 物流

此政策下，所有cookies將需要清楚標示Samesite值,

若未標示,Chrome會強制將Samesite指定為LAX

若您的網站未指定Samesite=None將可能出現無法預期的錯誤。

Samesite=Strict Samesite=LAX Samesite=None (允許跨網站存取)

例如:結帳完被登出、物流電子地圖無法將資料回寫

如何自行檢測?
開啟您的網站>Ctrl+Shift+I

如出現下圖表示未設置，預設為(LAX)

（請注意，在Chrome 85之前，每次加載頁面時，DevTools中都會向JavaScript控制台發送有關不兼容Cookie的消息。在Chrome 85中，這些消息已從控制台中刪除。)


自家網站的調整方式
1.使用HTTPS 加強交易雙方安全

2.將Set-Cookie 的行為都加上 SameSite 的屬性

SameSite=None + HTTPS：允許跨網域存取，但是若沒 HTTPS 則預設「拒絕」跨網域存取
無效：Set-Cookie: remote_session=abc123; SameSite=None
有效：Set-Cookie: remote_session=abc123; SameSite=None; Secure

PHP 7.2 以前的 SameStie Cookie 寫法
header(‘Set-Cookie: cross-site-cookie=name; SameSite=None; Secure’);

header(‘Set-Cookie: cookie2=name; SameSite=None; Secure’, false);

PHP 7.3 以後的 SameStie Cookie 寫法
setcookie(‘cross-site-cookie’, ‘name’, [‘samesite’ => ‘None’, ‘secure’ => true]);

setcookie（’cookie2’，’name’，[‘samesite’=>’None’，’secure’=> true]）;

設定 .htaccess
Header always edit Set-Cookie ^(.)$ “$1;HttpOnly;Secure;SameSite=None”
設定 Apache httpd.conf
Header always edit Set-Cookie ^(.)$ “$1;HttpOnly;Secure;SameSite=None”

設定完成，請記得重啟服務及清除瀏覽記錄。

網站還沒調整好，消費者端暫時的解決方式
Step 1. 於 Chrome 瀏覽器網址列輸入 chrome://flags
Step 2. 搜尋Samesite
Step 3. 將以下三個設定設定為Disabled
Step 4. 按下Relaunch,重啟瀏覽器

Safari

關閉 防止跨網站的追蹤功能>重啟瀏覽器

https://blog.chromium.org/2020/02/samesite-cookie-changes-in-february.html

綠界 金流-AIO出現MerchantID is Null

發表於 2020-09-04 更新於 2020-09-14 分類於 金流

若IOS行動裝置無法結帳，出現MerchantID is Null 畫面:


原因是消費者使用IOS行動裝置中的 Facebook 、LINE等APP 直接開啟您的商店連結進行結帳，

因為APP無法接受開新分頁或另開視窗以致表單資料無法POST給綠界故無法完成交易。

若您的系統設計是使用開新分頁或另開視窗，建議您將Form 表單的屬性 target=”_blank”移除

或提示消費者將您的商店連結複製到手機瀏覽器上進行交易。

綠界 模組-woocommerce建立物流訂單按鈕未出現常見原因

發表於 2020-09-04 更新於 2020-09-07 分類於 模組

後台未出現建立物流訂單按鈕及變更門市按鈕，情境一

原因:寄件人名稱為空

請至WooCommerce>設定>運送方式>綠界科技超商取貨，修改寄件人名稱

後台未出現建立物流訂單按鈕及變更門市按鈕，情境二
https://www.ecpay.com.tw/Upload/Announce/201811/images/647b920181114110725.jpg
原因:寄件人名稱超過10字元(5個中文字或10個英文字)

請至WooCommerce>設定>運送方式>綠界科技超商取貨，修改寄件人名稱

後台未出現建立物流訂單按鈕及變更門市按鈕，情境三

原因:使用萊爾富C2C、統一超商C2C未設定寄件人手機

請至WooCommerce>設定>運送方式>綠界科技超商取貨，填入寄件人手機

綠界 金流-站內付常見問題

發表於 2020-09-04 更新於 2020-09-07 分類於 金流

站內付常見問題
綠界科技為第三方支付公司，站內付模式為iframe框架所構成，行動裝置上會以開新視窗方式進行交易

Q:為什麼按下付款按鈕沒有反應?

A:請確認您的瀏覽器是否阻擋開新視窗

Q:為什麼卡號填完以後出現”交易已逾時，請重新交易”

A:請確認您的瀏覽器cookie設置是否封鎖第三方cookie/禁止第三方網站儲存及讀取cookie資料

Q:為什麼付款頁面視窗這麼小?

A:站內付模式外再包一層iframe是允許的，但請開發者注意是否已調整成為正確且合適的大小

Q:為什麼填完信用卡號碼送出後會出現此畫面?

A:請依以下步驟進行確認

1.網路是連線否正常 2.此張信用卡是否支援3D驗證功能

上述步驟檢查完畢，仍出現相同畫面

請與技術支援中心聯繫，謝謝。

綠界 金流-信用卡授權等於扣款成功嗎?

發表於 2020-09-04 更新於 2021-10-05 分類於 金流

當信用卡消費產生時會進行信用卡授權，此時會佔用卡片上的可用額度，但實際上並非已請/扣款。

也就是說，當消費者收到消費簡訊通知、APP通知時


待商家進行關帳動作後，綠界才會協助將資料送給銀行請款。此時才算完成扣款動作喔!

Q:為什麼訂單顯示『3D授權未於時間內完成，故交易失敗』，消費者打去發卡銀行問，銀行的回覆卻是授權成功?

A:因為綠界未收到銀行授權成功回覆，會將此單判定為失敗，判定失敗的訂單，將不會向銀行請款。

Q:明明交易失敗，為什麼帳單顯示這筆交易明細

A:因為各家銀行的通知方式或帳單顯示方式不同，可能會在”未請款”帳單中出現該筆交易，但不代表已扣款成功。
