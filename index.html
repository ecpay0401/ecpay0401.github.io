綠界-模組woocommerce選擇電子地圖遇到『找不到加密金鑰，請確認是否有申請開通此物流方式!』

發表於 2020-09-07 分類於 模組

使用Woocommerce串接綠界服務，選擇門市時遇到『找不到加密金鑰，請確認是否有申請開通此物流方式!』原因如下三點：

1.測試模式若勾選，必須使用測試專用的串接資訊，才能順利選擇門市：

B2C物流類型帳號

特店編號=2000132

HashKey=5294y06JbISpM5x9

HashIV=v77hoKGq4kWxNNIS

C2C物流類型帳號

特店編號=2000933

HashKey=XBERn1YOvpM9nfZc

HashIV=h1ONHk4P4yqbl5LK

反之，若未勾選，則必須使用正式申請的串接資訊，才能選擇門市：

請至廠商管理後台https://vendor.ecpay.com.tw/＞系統開發管理＞系統介接設定 就可取得

2.若需使用物流服務，尚需另外申請，請至https://www.ecpay.com.tw/IntroTransport/Index完成申請流程才能使用

3.待申請完成，請記得當初申請的物流類型是C2C 或 B2C，並設定於上圖的『物流類型』，設定與您申請的相符，才能使用物流服務。

綠界-電子發票 商品明細資料格式不正確

發表於 2020-09-07 分類於 電子發票

商品明細資料格式不正確

此錯誤訊息顯示商品明細『項次』對照有誤，請參考下方，每個參數都需有相同的項目數，若多/少一個就會遇到此錯誤。

ItemCount=1|2|1|1

ItemPrice=20|30|40|-20

ItemAmount=20|60|40|-20

ItemName=商品A|商品B|運費|折扣

ItemWord=個|個|筆|筆

綠界-電子發票 驗證發票金額發現錯誤，與商品合計金額不符的可能原因

發表於 2020-09-07 分類於 電子發票

驗證發票金額發現錯誤，與商品合計金額不符的可能原因

※請留意ItemAmount需統一為含稅金額，所以商品金額需符合以下規則。

1.當vat=1 ItemPrice(含稅)*ItemCount=ItemAmount(含稅) 各項總合並四捨五入=salesAmount(含稅)

EX：

ItemPrice=10|-20|30

ItemCount=2|1|2

ItemAmount=20|-20|60

salesAmount=60

2.當vat=0 ItemPrice(不含稅)ItemCount1.05=ItemAmount(含稅) 各項總合並四捨五入=salesAmount(含稅)

EX：

ItemPrice=2980|-566|1

ItemCount=2|1|1

ItemAmount=6258|-594.3|1.05

salesAmount=5665

綠界-金流 常見付款狀態未更新原因

發表於 2020-09-07 分類於 金流

常見付款狀態未更新原因
若消費者已經付款，綠界廠商管理後台也已更新付款

但您的系統後台(Woocommerec、Opencart、Shopline、QDM、美安….平台)

付款狀態未被更新，請程式人員檢查以下設定:

(1)接收付款通知的伺服器是否異常

● 如您正在舉辦搶購活動，請務必確保伺服器可短時間大量接收綠界回傳的付款結果通知

Ex:AWS、台灣固網….等其他網路業者，預設防火牆機制，短期間接收大量回傳會視為攻擊事件，請將綠界IP加入白名單

(2)近期是否有變更接收付款通知的ReturnURL網址

(3)接收付款結果的ReturnURL程式是否有判斷交易成功(RtnCode=1表示付款成功，其餘代碼為失敗)

(4)貴司系統收到綠界回傳結果，是否有正確回應1|OK

(5)因資安政策規定綠界僅提供443 PORT且需使用合法的DomainName進行設定

上述原因確認完畢，仍然無法排除此問題請提供以下資訊

並Email至techsupport@ecpay.com.tw

(1)商店代號(MerchantID)

(2)廠商交易編號(MerchantTradeNo)

(3)交易日期(MerchantTradeDate)

綠界-金流 綠界帳戶餘額低於退刷金額或為負數導致無法退刷及建立物流訂單

發表於 2020-09-07 分類於 金流

若您的帳戶餘額低於退刷金額為負數

將無法進行退刷以及建立物流訂單

請至綠界廠商管理後台進行預付動作，

如需進行退刷作業，服務項目請選擇”信用卡退刷款” 請存入足夠退刷的金額

如需進行物流費用預存，服務項目請選擇”預付物流款”請存入足夠每周二結算物流費用使用金額。

發表於 2020-09-07 分類於 金流

當建立(金流、物流、電子發票)訂單失敗且出現訊息：
●SSL certificate problem: certificate has expired
●SSL certificate problem: unable to get local issuer certificate
●sun.security.validator.ValidatorException: PKIX path building failed:
sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target


表示您的伺服器(server)根憑證無法自動更新出現此錯誤，請下載根憑證後手動更新：

https://crt.sh/?d=1720081

檔名: 1720081.crt

如需更多說明請參考以下文章

綠界 模組woocommerce交易失敗！商店尚未開啟收款服務

發表於 2020-09-04 更新於 2020-09-07 分類於 模組

此錯誤顯示您的『模式』與設定的『MerchantID、HashKey、HashIV』搭配有誤，請參考下述調整。

測試模式必須使用測試專用的MerchantID、HashKey、HashIV，並勾選測試模式，才能順利建立訂單。


正式模式必須使用正式申請的MerchantID、HashKey、HashIV，並取消勾選測試模式，才能順利建立訂單

綠界 金流-AIO出現MerchantID is Null

發表於 2020-09-04 更新於 2020-09-07 分類於 金流

若IOS行動裝置無法結帳，出現MerchantID is Null 畫面:


原因是消費者使用IOS行動裝置中的 Facebook 、LINE等APP 直接開啟您的商店連結進行結帳，

因為APP無法接受開新分頁或另開視窗以致表單資料無法POST給綠界故無法完成交易。

若您的系統設計是使用開新分頁或另開視窗，建議您將Form 表單的屬性 target=”_blank”移除

或提示消費者將您的商店連結複製到手機瀏覽器上進行交易。

綠界 模組-woocommerce建立物流訂單按鈕未出現常見原因

發表於 2020-09-04 更新於 2020-09-07 分類於 模組

後台未出現建立物流訂單按鈕及變更門市按鈕，情境一

原因:寄件人名稱為空

請至WooCommerce>設定>運送方式>綠界科技超商取貨，修改寄件人名稱

後台未出現建立物流訂單按鈕及變更門市按鈕，情境二
https://www.ecpay.com.tw/Upload/Announce/201811/images/647b920181114110725.jpg
原因:寄件人名稱超過10字元(5個中文字或10個英文字)

請至WooCommerce>設定>運送方式>綠界科技超商取貨，修改寄件人名稱

後台未出現建立物流訂單按鈕及變更門市按鈕，情境三

原因:使用萊爾富C2C、統一超商C2C未設定寄件人手機

請至WooCommerce>設定>運送方式>綠界科技超商取貨，填入寄件人手機
