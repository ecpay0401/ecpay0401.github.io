綠界-電子發票-我適合哪種開立發票方式?

發表於 2021-10-05 分類於 電子發票

主要開立對象為一般消費者（Business to Customer）或營業人

B2C 存證發票

如為一般消費者，不需要輸入買方統編，可列印出來交付給買受人，或是將發票資料存入載具。

如為營業人，開立時需輸入買方統編，可以不用列印紙本發票，只要透過ECPAY電子郵件寄送電子發票證

明聯檔案給買方公司即可。

主要開立對象為營業人（Business to Business）或政府機關、財團法人

B2B 存證發票

開立發票時輸入買方統編，可以不用列印紙本發票，只要透過ECPAY電子郵件寄送電子發票證明聯檔案給買方公司即可。

B2B 交換發票

開立發票時需要雙方進行確認，僅資料交換。

營業人A開立電子發票給營業人B，營業人B必須接收與檢視發票內容並做確認或退回。

比較表:

Q:電子發票可以開三聯式發票(B2B)嗎?

A:可以，傳統的三聯式發票會多一聯存根聯，以利同為營業人的買方做為存根之用。

而電子發票若買方同為營業人，則可以電子形式傳送發票資料檔，供買方營業人後續申報營業稅扣抵之用。

Q:可以串接一個規格(API)同時開立B2C和有統一編號(B2B)發票嗎?

A:可以,若您的主要開立對象B2C大於B2B建議參考電子發票規格書 ，當統一編號(CustomerIdentifier)有值時即為B2B發票

綠界-電子發票服務優化2021/8/24(二)

發表於 2021-10-05 分類於 電子發票

因應電子發票服務優化，綠界將於2021/8/24(二)調整系統架構，調整後主機 IP 將會不固定。

To enhance ECPay’s E-invoice services, from 2021/08/24th (Tue.) ECPay will change the server IP from static to dynamic.

Please notice:

1.如您是使用綠界廠商管理後台開立發票，請忽略本通知。

If you are using ECPay’s admin portal to issue/create the e-invoice, please ignore this notification.

2.如貴司原本防火牆沒有鎖定綠界 IP，也請忽略本通知。

If your firewall never set the ECPay’s IP in the access list of firewall policy, please ignore this notification as well.

3.如貴司防火牆需設定開通綠界 IP，請改以 FQDN(Fully Qualified Domain Name)方式鎖定

If your firewall needs to set ECPay’s IP into the access list of firewall policy, please set it in the way of FQDN (fully qualified domain name.)

允許連出(Allow outbound connection)：

einvoice.ecpay.com.tw TCP 443、einvoice-stage.ecpay.com.tw TCP 443

允許連入(Allow inbound connection)：

postgate.ecpay.com.tw TCP 443、postgate-stage.ecpay.com.tw TCP 443

若您的主機是使用托管方式，請將此訊息轉知主機商。

綠界-金流-錯誤代碼:10200141商店未開啟收款服務

發表於 2021-10-05 分類於 金流

金流:商店未開啟收款服務原因
1.請確認您的API網址，

若為https://payment-stage.ecpay.com.tw/Cashier/AioCheckOut/V5

表示在測試環境，需使用測試環境的 MerchantID、HashIV、HashKey

MerchantID應填入 2000132

HashKey必須為5294y06JbISpM5x9

HashIV必須為v77hoKGq4kWxNNIS

2.若為正式環境 API網址為https://payment.ecpay.com.tw/Cashier/AioCheckOut/V5

還是出現此錯誤訊息則表示未完成金流服務申請，需與客服中心聯繫

3.若服務申請皆已通過且API網址已調為正式環境，出現以下訊息

請確認使用的MerchantID是否為正式，正式資料查詢請參考MerchantID及HashKey、HashIV的取得

Why receiving this error: “Message code: 10200141”?

If you call ECPay’s payment API (“…/Cashier/AioCheckOut/V5”) and find an error says “訊息代碼 Message code：10200141

訊息說明 Note for message：商店尚未提供任何付款方式，請聯繫商店處理。”, following are the reason why and how you can solve this problem.

  1. If the API URL you called is on ECPay’s stage environment (i.e. testing mode, with an “-stage “ in the URL) but you use the MerchantID, HashKey and HashIV of production environment (live mode).

→This is due to that the environment (i.e. the API URL) is not complied with the MerchantID and its pair of HashKey and HashIV.

In this case, firstly, please check your API URL. If the API URL has “-stage”, you should use the testing MerchantID and its pair of HashKey and HashIV (i.e. MerchantID=2000132, HashKey= 5294y06JbISpM5x9 and HashIV= v77hoKGq4kWxNNIS).

  1. If the MerchantID and its pair of HashKey and HashIV you used are for production and you already also confirm that the API URL has no “-stage” (i.e. the live mode), but this error code happened still.

→This is due to that you (as a merchant) have not yet completed the application of payment service to ECPay. In this case, please contact ECPay’s customer service center to check the application status.

  1. If you already confirm the above and call the API but another error shows: “訊息代碼 Message code：10200094

訊息說明 Note for message：請檢查是否於正式環境誤用測試環境之MerchantID”

→This is due to that you call the API of production environment (live mode) but incorrectly use the MerchantID, HashKey and HashIV of stage environment (testing mode). In this case please check your MerchantID, HashKey, and HashIV. To retrieve it,

please see this FAQ: How can I know my MerchantID, HashKey, and HashIV through ECPay’s admin (panel)?

https://www.ecpay.com.tw/CascadeFAQ/CascadeFAQ_Qa?nID=1179

綠界-金流-錯誤代碼:10300023 本次交易未提供任何付款方式

發表於 2021-10-05 分類於 金流

綠界錯誤代碼:10300023 本次交易未提供任何付款方式
若您的收款功能突然無法使用且收到此錯誤代碼

表示收款審核不通過或服務未被開通

建議您可先至會員專區訊息中心或收款審核資料處確認收款服務狀態。

確認收款審核狀態步驟 綠界首頁 >會員專區>收款審核資料

不通過原因請於訊息中心確認 綠界首頁 >會員專區>訊息中心

若您對於審核結果有疑慮，請與客服中心連繫

綠界-金流-為什麼Key完卡號送出以後一片空白

發表於 2021-03-24 分類於 金流

為什麼Key完卡號送出以後一片空白?

1.在ECPAY輸入完卡號送出後會將頁面導向各銀行進行3D驗證，

如您使用的瀏覽裝置版本不符合銀行規範(請自行參考各銀行規範)，將可能未知出現錯誤

建議更新瀏覽器版本或更換裝置重新下單購買。

2.若是使用webview方式與Ecpay串接，請注意以下事項

UserAgent 設定:若有指定版本可能會造成在銀行端進行3D驗證時出現以下畫面

因安全性考量如必須要指定版本，建議定期更新，避免無法進行3D驗證而損失訂單。

發表於 2020-09-07 更新於 2020-09-14 分類於 模組

綠界-模組woocommerce選擇電子地圖遇到『找不到加密金鑰，請確認是否有申請開通此物流方式!』

發表於 2020-09-07 分類於 模組

使用Woocommerce串接綠界服務，選擇門市時遇到『找不到加密金鑰，請確認是否有申請開通此物流方式!』原因如下三點：

1.測試模式若勾選，必須使用測試專用的串接資訊，才能順利選擇門市：

B2C物流類型帳號

特店編號=2000132

HashKey=5294y06JbISpM5x9

HashIV=v77hoKGq4kWxNNIS

C2C物流類型帳號

特店編號=2000933

HashKey=XBERn1YOvpM9nfZc

HashIV=h1ONHk4P4yqbl5LK

反之，若未勾選，則必須使用正式申請的串接資訊，才能選擇門市：

請至廠商管理後台https://vendor.ecpay.com.tw/＞系統開發管理＞系統介接設定 就可取得

2.若需使用物流服務，尚需另外申請，請至https://www.ecpay.com.tw/IntroTransport/Index完成申請流程才能使用

3.待申請完成，請記得當初申請的物流類型是C2C 或 B2C，並設定於上圖的『物流類型』，設定與您申請的相符，才能使用物流服務。

綠界-電子發票 商品明細資料格式不正確

發表於 2020-09-07 分類於 電子發票

商品明細資料格式不正確

此錯誤訊息顯示商品明細『項次』對照有誤，請參考下方，每個參數都需有相同的項目數，若多/少一個就會遇到此錯誤。

ItemCount=1|2|1|1

ItemPrice=20|30|40|-20

ItemAmount=20|60|40|-20

ItemName=商品A|商品B|運費|折扣

ItemWord=個|個|筆|筆

綠界-電子發票 驗證發票金額發現錯誤，與商品合計金額不符的可能原因

發表於 2020-09-07 分類於 電子發票

驗證發票金額發現錯誤，與商品合計金額不符的可能原因

※請留意ItemAmount需統一為含稅金額，所以商品金額需符合以下規則。

1.當vat=1 ItemPrice(含稅)*ItemCount=ItemAmount(含稅) 各項總合並四捨五入=salesAmount(含稅)

EX：

ItemPrice=10|-20|30

ItemCount=2|1|2

ItemAmount=20|-20|60

salesAmount=60

2.當vat=0 ItemPrice(不含稅)ItemCount1.05=ItemAmount(含稅) 各項總合並四捨五入=salesAmount(含稅)

EX：

ItemPrice=2980|-566|1

ItemCount=2|1|1

ItemAmount=6258|-594.3|1.05

salesAmount=5665
